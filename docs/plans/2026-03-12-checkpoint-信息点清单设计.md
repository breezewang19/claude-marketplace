# CheckPoint Architect - 推荐信息点清单功能设计

**版本**: 1.0
**日期**: 2026-03-12
**状态**: 已批准

---

## 1. 需求概述

在生成审查点内容的同时，自动推断并输出推荐的信息点清单，帮助用户了解需要从各文书中抽取的关键字段。

### 1.1 核心需求

| 需求项 | 说明 |
|--------|------|
| 输出位置 | 生成内容后显示 + 输出文件中都有 |
| 生成方式 | 混合模式（关键词匹配 + 逻辑反推） |
| 标注格式 | 「时间名称」 |
| 展示位置 | 独立段落，审查步骤之后 |
| 文书分组 | 按文书分组 |
| 空值处理 | 只输出能推断出的信息点 |
| 免责声明 | 放在文件结尾 |

---

## 2. 功能设计

### 2.1 信息点推断引擎

采用混合模式进行推断：

#### 2.1.1 关键词匹配

从用户描述的审查逻辑中识别以下关键词模式：

| 关键词类型 | 识别关键词 | 推断信息点 |
|------------|------------|------------|
| 时间类 | 时间、时长、期限、日期间隔 | 「日期」「时间」「时长」 |
| 编号类 | 编号、号码、代码 | 「编号」「号码」 |
| 人员类 | 签名、盖章、审批、负责人 | 「签名」「盖章」「审批人」 |
| 金额类 | 金额、数目、款项 | 「金额」「数目」 |
| 状态类 | 状态、结果、是否通过 | 「状态」「结果」 |
| 地点类 | 地点、地址、场所 | 「地点」「地址」 |

#### 2.1.2 逻辑反推

基于审查逻辑中的判断条件反向推导：

| 判断条件 | 推断信息点 |
|----------|------------|
| 超过X小时/天 | 「时长」「起始时间」「结束时间」 |
| 一致/不一致 | 「比对字段1」「比对字段2」 |
| 是否存在 | 「文书名称」「审批记录」 |
| 是否齐全 | 「清单字段1」「清单字段2」 |

### 2.3 实现位置

修改 `checkpoint-architect/SKILL.md` 的以下部分：

1. **Step 4: Generate Content** - 在生成审查步骤时，同步推断信息点
2. **Step 5: Generate Logic Diagram** - 保持不变
3. **Step 6: User Review** - 将信息点清单作为参考展示给用户
4. **Step 8: Generate Output File** - 将信息点清单和免责声明写入文件
