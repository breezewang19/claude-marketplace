# Checkpoint Architect - 批量模式设计文档

**日期：** 2026-03-20
**版本：** 1.0
**作者：** yrwang45

---

## 背景

checkpoint-architect skill 目前支持两种模式：
- **A. 快速模式** - 用自然语言描述审查逻辑
- **B. 引导模式** - 基于模板的引导式问答

用户有时需要处理一批已由民警梳理好逻辑的审查点草稿文件，逐一使用 A/B 模式效率较低。因此新增批量模式，支持无交互自动批量生成。

---

## 需求

- 用户指定一个目录，批量处理该目录下所有 `.md` 文件
- 每个文件包含半结构化草稿或自由文本笔记（格式不规范，需 AI 整理）
- 无需用户交互，AI 自主完成解析和生成
- 失败的文件跳过并记录，最终输出汇总报告
- 输出文件统一写入现有 `output/` 目录
- 在模式选项中提示用户：若追求更好效果，建议使用 A 或 B 模式

---

## 设计

### Step 3 模式选择变更

在现有 A、B 两个选项后新增选项 C：

```
C. 批量模式（Batch Mode）
- 适合：已有一批逻辑基本清晰的审查点草稿文件
- 方式：指定目录，AI 自动解析并批量生成，无需交互
- 💡 提示：批量模式适合快速处理大量草稿，但 AI 自动解析可能存在偏差。
         若对准确性要求较高，建议使用快速模式（A）或引导模式（B）逐一处理。
```

### 批量模式执行流程

```
用户选择 C
  │
  ├─ 提示用户输入目录路径
  │
  ├─ 扫描目录下所有 .md 文件
  │   └─ 若无文件，提示并退出
  │
  ├─ 逐个文件处理（循环）
  │   ├─ 解析文件内容
  │   │   ├─ 提取：审查点名称
  │   │   ├─ 提取：文书类型
  │   │   └─ 提取：审查逻辑步骤
  │   │
  │   ├─ 解析成功 → 执行生成流程
  │   │   ├─ Step 4 逻辑（快速模式路径，跳过交互）
  │   │   ├─ Step 5 生成流程图
  │   │   ├─ Step 7 验证
  │   │   └─ Step 8 输出文件到 output/
  │   │
  │   └─ 解析失败 → 记录失败原因，跳过，继续下一个
  │
  └─ 输出汇总报告
```

### 文件解析规则

对于半结构化草稿和自由文本笔记，AI 解析时遵循以下优先级：

1. **审查点名称**：查找包含"是否"、"有无"、"合规"、"检查"等关键词的短语；若无明显标识，取文件第一行或文件名
2. **文书类型**：查找《》书名号包裹的内容，或包含"证"、"书"、"表"、"单"等文书关键词的词组
3. **审查逻辑**：提取描述审查步骤、判断条件、结论的文字，转换为标准的"第一步/第二步/第三步"结构

解析失败条件：
- 无法识别审查点名称
- 无法识别任何文书类型
- 内容为空或无意义

### 汇总报告格式

```
批量处理完成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 成功：X 个
❌ 失败：X 个（已跳过）

【成功列表】
- output/2026-03-20-审查点名称A.md
- output/2026-03-20-审查点名称B.md

【失败列表】
- input/文件名.md - 原因：无法识别审查点名称

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 对于失败的文件，建议使用快速模式（A）或引导模式（B）手动处理。
```

## 实现范围

**需要修改的文件：**
- `skills/checkpoint-architect/SKILL.md`
  - Step 3：新增选项 C 及说明
  - 新增 Step 4C（批量模式处理逻辑）
  - Step 9：新增批量模式的汇总报告展示

**不需要修改的文件：**
- `checkpoint-templates.md` - 批量模式不使用引导模板
- `checkpoint-validator.md` - 复用现有验证逻辑

---

## 约束

- 批量模式不支持交互式修改，用户若需调整某个审查点，需单独用 A/B 模式重新处理
- 输出目录固定为 `output/`，与现有模式保持一致
- 仅处理 `.md` 文件，忽略其他格式
