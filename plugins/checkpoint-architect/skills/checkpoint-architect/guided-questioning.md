# Guided Questioning - 引导模式提问框架

## 用途
为 Step 4B（引导模式）提供逐步提问、回答质量评估和问题推进逻辑。

## 核心规则: 一次一问

- 每次只问一个问题
- 收到完整回答后才进入下一题
- 绝不同时呈现多个问题

**提问格式:**
```
【问题 X/总数】{question_text}

{提示或示例}
```

## 回答质量评估

**1. Complete Answer → 进入下一题**
- 包含所有请求信息，具体且清晰

**2. Incomplete Answer → 追问细节**
- 缺少关键信息、过于简略
- 策略: "感谢你的回答！还需要补充：你提到了 {partial}，还需确认：[缺失项]"

**3. Unclear Answer → 请求澄清**
- 含糊、矛盾、使用模糊词
- 策略: "我想确保理解正确。你的意思是：[我的理解]。如果不是，能否换个方式说明？"

**4. Off-topic Answer → 引回正题**
- 未回答当前问题
- 策略: "我注意到你提到了 {off_topic}。让我们先完成当前问题：【问题 X/总数】{original_question}"

## 模糊回答处理

对于含"差不多"、"大概"、"可能"的回答:
```
我理解你的大致想法，但为了生成准确的审查点，我需要更具体的信息。

例如，你提到"{vague_answer}"，能否具体说明：
- [需要明确的点1]
- [需要明确的点2]
```

## 问题推进规则

**进入下一题条件:** 回答完整、清晰、无歧义

**留在当前题条件:** 回答模糊、不完整、需澄清

**允许回退:** 用户说"上一个问题"或"返回"时，回到上一题并展示之前的答案供修改

## 示例

**Example 1: 处理模糊回答**
```
Q: 【问题 2/5】每类文书需要检查哪些必备要素？

User: "差不多就是常见的那些要素"

System: 我理解你的大致想法，但为了生成准确的审查点，我需要更具体的信息。
例如，你提到"常见的那些要素"，能否具体说明：
- 对于《受案登记表》，需要检查哪些要素？（如：受案编号、受案时间、审批人签名等）
- 对于《立案决定书》，需要检查哪些要素？（如：案号、立案时间、公章等）

User: 受案登记表要检查受案编号、受案时间、审批人签名。立案决定书要检查案号、立案时间、公章。

System: 非常好！信息很清晰。让我们继续下一个问题。
【问题 3/5】如何判断要素是否完整？
```

**Example 2: 处理不完整回答**
```
Q: 【问题 3/5】如何判断要素是否完整？

User: "字段为空"

System: 感谢你的回答！还需要补充：
你提到了"字段为空"，还需确认：
- 除了字段为空，还有其他判断标准吗？（如：未签名、无公章、内容不完整等）
- 对于签名和公章，如何判断是否缺失？

User: 还有未签名、无公章。签名和公章如果没有就算缺失。

System: 明白了！让我们继续。
【问题 4/5】是否需要检查签名/公章？
```

## 信息汇总与确认

所有问题回答完毕后，汇总确认:
```
太好了！我已收集到所有信息。让我总结一下：

【审查点信息汇总】
- 审查点名称：{checkpoint_name}
- 文书类型：{document_types}
- 审查模式：{pattern_name}

【问题回答汇总】
1. {question_1}: {answer_1}
2. {question_2}: {answer_2}
...

信息是否准确？如需修改，请告诉我具体要改哪一项。
确认无误后，我将生成审查点内容。
```

用户确认后，使用模板步骤结构生成内容，确保包含文书缺失处理和明确判定结论。
