# CheckPoint Templates - 审查点模板库

## 用途
为产品经理和运营经理提供5种常见审查点模式的模板、引导问题和示例，帮助快速构建符合规范的审查点内容。

## 模板概览

本模板库包含以下5种审查模式：

1. **文书完整性检查** - 检查文书是否存在、要素是否齐全
2. **时限合规性检查** - 涉及日期计算和期限比对
3. **信息一致性检查** - 跨文书字段比对
4. **程序合规性检查** - 检查流程是否符合法定程序
5. **自定义逻辑** - 完全自由的审查逻辑

---

## 模板 1: 文书完整性检查

### 适用场景
- 检查某类文书是否存在于卷宗中
- 检查文书中的必备要素是否齐全（如签名、公章、编号等）
- 检查文书内容是否完整填写

### 典型案例
- 受案登记表规范性检查
- 立案决定书要素完整性检查
- 传唤审批程序检查（是否有呈请报告书）

### 引导问题清单

请回答以下问题，我将帮你生成审查点内容：

1. **需要检查哪些文书？**
   - 例如：《受案登记表》、《立案决定书》
   - 提示：列出所有需要检查的文书类型

2. **每类文书需要检查哪些必备要素？**
   - 例如：受案编号、受案时间、公章、审批人签名
   - 提示：分文书类型列出要素清单

3. **如何判断要素是否完整？**
   - 例如：字段为空、未签名、无公章
   - 提示：明确"缺失"的判断标准

4. **是否需要检查签名/公章？**
   - 是/否
   - 如果是，具体检查哪些签名/公章？

5. **发现要素缺失时如何处理？**
   - 例如：判定为"存在问题"，列明缺失要素名称及所在文书编号
   - 提示：明确问题描述方式

### 步骤结构模板

```
第一步：识别审查对象。
识别卷宗中所有《[文书名称A]》及《[文书名称B]》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。

第二步：逐一审查文书要素完整性。
针对每一类文书，执行以下操作：
a. 核查《[文书名称A]》要素：
 - 是否具备[要素1]、[要素2]、[要素3]
 - 是否完整填写[要素4]、[要素5]
 - 是否有[签名/公章]
b. 核查《[文书名称B]》要素：
 - 是否具备[要素1]、[要素2]
 - 是否含有[要素3]
 - 是否有[签名/公章]

第三步：要素缺失判断。
出现以下任一情况即判定为要素缺失：
a. 任何一份文书缺少上述必备要素
b. 要素填写不完整（如[具体示例]）
c. 要素存在明显错误（如[具体示例]）
d. 签章不全或缺少必要的审批流程记录

第四步：得出结论。
- 若所有文书要素齐全、填写规范，则判定为"不存在问题"
- 若任何一份文书存在要素缺失情况，则整体判定为"存在问题"，并应具体列明缺失要素名称及所在文书编号，要求补充完善相关手续。
```

### 示例输出

**审查点名称：** 受、立案文书规范性检查

**审查步骤：**
```
第一步：识别审查对象。识别卷宗中所有《受案登记表》及《立案决定书》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。
第二步：逐一审查文书要素完整性。针对每一类文书，执行以下操作：
a. 核查《受案登记表》要素：
 - 是否具备受案编号、受案时间、案件来源、报案人姓名、性别、身份证号、接报地点、接报单位
 - 是否完整填写简要案情、受案审批人签名
 - 是否有公章
b. 核查《立案决定书》要素：
 - 是否具备立案编号、立案时间、案件名称
 - 是否含有相关的法律依据
 - 是否有公章
第三步：要素缺失判断。出现以下任一情况即判定为要素缺失：
a. 任何一份文书缺少上述必备要素
b. 要素填写不完整（如仅有罪名无法律依据条款）
c. 要素存在明显错误（如时间逻辑矛盾、编号重复）
d. 签章不全或缺少必要的审批流程记录
第四步：得出结论。
- 若所有文书要素齐全、填写规范，则判定为"不存在问题"
- 若任何一份文书存在要素缺失情况，则整体判定为"存在问题"，并应具体列明缺失要素名称及所在文书编号，要求补充完善相关手续。
```

## 模板 2: 时限合规性检查

### 适用场景
- 检查某个时间间隔是否符合法定期限
- 计算两个日期之间的天数/小时数
- 判断是否超时或延期

### 典型案例
- 传唤时长是否合规（不超过12/24小时）
- 是否及时受案（报案到受案的间隔）
- 羁押期限是否超期

### 引导问题清单

请回答以下问题，我将帮你生成审查点内容：

1. **需要比对哪些日期/时间？**
   - 例如：报案时间 vs 受案日期、到达时间 vs 离开时间
   - 提示：明确起始时间和结束时间

2. **这些日期/时间从哪些文书中提取？**
   - 例如：《询问笔录》中的报案时间、《受案登记表》中的受案日期
   - 提示：指明文书名称和字段名称

3. **需要计算什么时间间隔？**
   - 例如：天数、小时数
   - 提示：明确计算单位和方法

4. **法定期限是多少？**
   - 例如：不超过1天、不超过12小时
   - 提示：提供具体的期限标准

5. **超时如何处理？**
   - 例如：判定为"存在问题"，指明超时时长
   - 提示：明确问题描述方式

6. **是否有延期文书？**
   - 是/否
   - 如果是，延期文书名称是什么？如何判断延期是否合规？

### 步骤结构模板

```
第一步：定位核心文书。
在卷宗中查找《[文书名称A]》和《[文书名称B]》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。

第二步：提取关键时间信息。
从《[文书名称A]》中提取"[时间字段A]"（精确到[年月日/时分]）。
从《[文书名称B]》中提取"[时间字段B]"（精确到[年月日/时分]）。

第三步：计算时间间隔。
计算从"[时间字段A]"到"[时间字段B]"的时间间隔（以[天数/小时]为单位）。

第四步：判断合规性。
- 若时间间隔在[X]以内（含），则判定为"不存在问题"
- 若时间间隔超过[X]但未超过[Y]，则检查是否存在《[延期文书名称]》
  - 若存在延期文书，则判定为"不存在问题"
  - 若不存在延期文书，则判定为"存在问题"，并指明超时时长
- 若时间间隔超过[Y]，则直接判定为"存在问题"，并指明超时时长

第五步：得出结论。
输出明确的判定结果，如判定为"存在问题"，应在结论中指明具体的文书编号及超时情况。
```

### 示例输出

**审查点名称：** 传唤时长是否合规

**审查步骤：**
```
第一步：识别审查对象。
确认当前涉案人员是否为犯罪嫌疑人。如果是，则从该人员的文书库中查找其所有的《传唤证》及对应的《呈请延长传唤报告书》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。
第二步：审查传唤时限。
针对该人员的每一份《传唤证》，执行以下操作：
a. 提取关键时间：从《传唤证》中提取"被传唤人到达时间"和"被传唤人离开时间"。
b. 计算传唤总时长：计算从"被传唤人到达时间"到"被传唤人离开时间"的总时长（以小时为单位）。
c. 判断合规性：
 - 若总时长超过12小时但未超过24小时，则检查是否存在对应的《呈请延长传唤报告书》。
 - 若总时长超过24小时，则直接判定为当前人员"存在问题"，并应指明具体问题。
第三步：得出结论。
- 若该人员的任何一次传唤出现以下情形，则判定为当前人员"存在问题"，并需指明该人员具体传唤证文号及超时时长：
 1. 总时长超过12小时但未超过24小时，且未找到对应的《呈请延长传唤报告书》。
 2. 总时长超过24小时。
- 否则，判定为当前人员"不存在问题"。
```

## 模板 3: 信息一致性检查

### 适用场景
- 跨文书比对同一信息是否一致
- 检查关键字段是否匹配
- 发现信息矛盾或冲突

### 典型案例
- 传唤地点与讯问地点是否一致
- 受案登记表与立案决定书中的案件信息是否一致
- 不同笔录中被害人陈述是否一致

### 引导问题清单

请回答以下问题，我将帮你生成审查点内容：

1. **需要比对哪些文书？**
   - 例如：《传唤证》和《讯问笔录》
   - 提示：列出所有需要比对的文书

2. **需要比对哪些字段/信息？**
   - 例如：传唤地点 vs 讯问地点、案件名称、涉案金额
   - 提示：明确字段名称和比对内容

3. **如何判断信息是否一致？**
   - 例如：完全相同、实质相同、允许的差异范围
   - 提示：明确一致性标准

4. **发现不一致时如何处理？**
   - 例如：判定为"存在问题"，指明不一致的具体内容
   - 提示：明确问题描述方式

5. **是否有可接受的差异？**
   - 例如：地址简写、金额小数点差异
   - 提示：说明哪些差异可以接受

### 步骤结构模板

```
第一步：定位核心文书。
从卷宗中查找《[文书名称A]》和《[文书名称B]》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。

第二步：提取关键信息。
从《[文书名称A]》中提取"[字段A]"。
从《[文书名称B]》中提取"[字段B]"。

第三步：比对信息一致性。
核对《[文书名称A]》中的"[字段A]"与《[文书名称B]》中的"[字段B]"是否一致。
[可选：说明一致性判断标准，如完全相同、实质相同等]

第四步：得出结论。
- 若信息一致，则判定为"不存在问题"
- 若信息不一致，则判定为"存在问题"，并应指明具体的文书编号及不一致的内容
```

### 示例输出

**审查点名称：** 传唤地点是否合规

**审查步骤：**
```
第一步：识别审查对象。
确认当前涉案人员是否为犯罪嫌疑人。如果是，则从该人员的文书库中查找该人员所有的《传唤证》及与之关联的《讯问笔录》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。
第二步：审查传唤与讯问地点。
针对该人员的每一组相关联的《传唤证》和《讯问笔录》，执行以下操作：
a. 提取地点信息：从《传唤证》中提取"传唤地点"；从对应的《讯问笔录》中提取"讯问地点"。
b. 比对地点一致性：核对《传唤证》上载明的"传唤地点"与《讯问笔录》中记载的"讯问地点"是否一致。
c. 判断地点合规性：判断"传唤/讯问地点"是否符合《公安机关办理刑事案件程序规定》的要求，即"传唤/讯问地点"是否为公安机关执法办案场所、被传唤人所在市、县内的指定地点或其住处；核查是否存在在宾馆、酒店等非执法场所进行讯问的情形。
第三步：得出结论。
- 若该人员的任何一次传唤出现以下情形之一，则判定为当前人员"存在问题"，并需指明当前人员的传唤证文号及违规地点信息：
 1. 传唤地点与讯问地点不一致。
 2. 传唤/讯问地点不符合法定要求。
- 否则，判定为当前人员"不存在问题"。
```

## 模板 4: 程序合规性检查

### 适用场景
- 检查某个法定程序是否完整执行
- 验证程序步骤的先后顺序
- 检查程序性文书是否齐全

### 典型案例
- 实施传唤程序是否经审批
- 管辖是否符合法律规定
- 强制措施是否履行告知程序

### 引导问题清单

请回答以下问题，我将帮你生成审查点内容：

1. **需要检查哪个法定程序？**
   - 例如：传唤审批程序、管辖确认程序
   - 提示：明确程序名称

2. **该程序包含哪些步骤？**
   - 例如：提交申请 → 审批 → 签发文书
   - 提示：列出完整的程序步骤

3. **每个步骤需要哪些文书支撑？**
   - 例如：申请阶段需要《呈请报告书》，审批阶段需要审批人签名
   - 提示：明确每个步骤的文书要求

4. **如何判断程序是否违规？**
   - 例如：缺少审批文书、审批时间晚于执行时间
   - 提示：列出违规情形

5. **是否有特殊情况或例外？**
   - 例如：上级指定管辖、紧急情况下的简化程序
   - 提示：说明例外情况的处理方式

### 步骤结构模板

```
第一步：识别审查对象。
从卷宗中查找《[主要文书]》及与之对应的《[程序性文书]》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。

第二步：审查程序完整性。
针对每一份《[主要文书]》，执行以下操作：
a. 核查程序性文书：根据《[主要文书]》的文号或内容信息，查找对应的《[程序性文书]》。
b. 审查程序要素：核查《[程序性文书]》中的[要素1]、[要素2]、[要素3]。
c. 判断合规性：出现以下任一情形，则判定为"存在问题"：
 - 未找到对应的《[程序性文书]》
 - [程序性文书]缺少[必备要素]
 - [程序性文书]的[时间]晚于《[主要文书]》的[时间]

第三步：得出结论。
- 若所有程序性文书齐全且符合要求，则判定为"不存在问题"
- 若存在程序违规情形，则判定为"存在问题"，并应指明具体的文书编号及缺失的程序环节
```

### 示例输出

**审查点名称：** 实施传唤程序是否经审批

**审查步骤：**
```
第一步：识别审查对象。
确认当前涉案人员是否为犯罪嫌疑人。如果是，则从该人员的文书库中查找该人员所有的《传唤证》及与之对应的《呈请传唤报告书》。若缺少以上文书，则判定为"缺少文书材料"，并具体列明缺失的文书名称。
第二步：审查传唤审批程序。
针对该人员的每一份《传唤证》，执行以下操作：
a. 核查审批文书：根据《传唤证》的文号或内容信息，查找对应的《呈请传唤报告书》。
b. 审查审批要素：核查找到的《呈请传唤报告书》中的"是否有文书公章"，若"是否有文书公章"为"是"，则不缺少公章，否则缺少公章。
c. 判断合规性：出现以下任一情形，则判定当前人员"存在问题"，并应在审查结论中指明具体传唤证文号及缺失的审批环节：
 - 任何一份《传唤证》未找到对应的《呈请传唤报告书》。
 - 报告书缺少公章。
 - 报告书的审批时间晚于《传唤证》的签发时间。
否则，判定为当前人员"不存在问题"。
```

## 模板 5: 自定义逻辑

### 适用场景
- 以上4种模式都不适用的特殊审查点
- 需要复杂的多步骤判断
- 涉及多种审查模式的组合

### 典型案例
- 是否如实受案（需要综合多份笔录判断）
- 书面传唤的适用是否违规（需要判断犯罪嫌疑人身份）
- 复杂的证据链完整性审查

### 引导问题清单

请回答以下问题，我将帮你逐步构建审查点内容：

1. **这个审查点要检查什么？**
   - 用一句话描述审查目标
   - 提示：明确审查的核心问题

2. **需要用到哪些文书？**
   - 列出所有相关文书
   - 提示：包括主要文书和辅助文书

3. **审查的第一步是什么？**
   - 例如：定位文书、识别对象、提取信息
   - 提示：从最基础的操作开始

4. **接下来需要做什么？**
   - 逐步描述每个审查步骤
   - 提示：我会引导你逐步完善

5. **如何判断是否存在问题？**

   - 描述结论的表达方式
   - 提示：确保包含"存在问题"/"不存在问题"/"缺少文书材料"

## 使用指南

### 如何选择合适的模板？

1. **文书完整性检查** - 如果你的审查点主要是"检查某个文书是否存在"或"检查文书中的要素是否齐全"
2. **时限合规性检查** - 如果你的审查点涉及"计算时间间隔"或"判断是否超期"
3. **信息一致性检查** - 如果你的审查点需要"比对两个文书中的同一信息"
4. **程序合规性检查** - 如果你的审查点要"检查某个法定程序是否完整执行"
5. **自定义逻辑** - 如果以上都不适用，或需要组合多种审查模式

### 使用流程

1. **选择模板**：根据审查点的性质选择最合适的模板
2. **回答引导问题**：按照模板中的引导问题清单逐一回答
3. **生成初稿**：基于你的回答，使用步骤结构模板生成审查点内容
4. **参考示例**：对照示例输出，调整表达方式和细节
5. **验证规范**：使用checkpoint-validator验证内容是否符合格式要求

### 注意事项

1. **文书名称必须使用书名号**：《传唤证》而不是"传唤证"
2. **必须处理文书缺失情况**：每个审查点都要包含"若缺少以上文书，则判定为'缺少文书材料'"
3. **必须有明确的判定结论**：使用"判定为'存在问题'"、"判定为'不存在问题'"、"判定为'缺少文书材料'"
4. **步骤结构要清晰**：使用"第一步"、"第二步"等明确标记
5. **不要包含程序会自动添加的元素**：不要写"审查点 1:"、"审查方法 (提示):"、"审查所需文书类型:"

### 常见问题

**Q: 我的审查点涉及多种模式，应该选哪个模板？**
A: 选择"自定义逻辑"模板，它允许你灵活组合多种审查模式。

**Q: 模板中的[占位符]如何填写？**
A: 根据你的实际审查需求，将占位符替换为具体的文书名称、字段名称、判断标准等。

**Q: 生成的内容需要包含法律依据吗？**
A: 不需要。法律依据由Python程序从法律法规库中自动匹配，你只需提供审查步骤。

**Q: 如何确保生成的内容符合规范？**
A: 完成后使用checkpoint-validator进行验证，它会检查格式、结构和逻辑完整性。
