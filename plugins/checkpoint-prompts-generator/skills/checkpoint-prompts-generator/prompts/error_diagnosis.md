# 错误诊断与修复选项

当 Python 脚本执行失败时，根据错误类型提供以下中文描述和修复选项：

| 错误类型 | 中文描述 | 提供的修复选项 |
|----------|----------|----------------|
| 文件被占用 | "文件正在被其他程序使用，无法读取" | 1) 关闭文件后重试 2) 换一个文件名 |
| 权限不足 | "没有写入输出目录的权限" | 1) 使用管理员权限运行 2) 指定其他输出目录 |
| 文件格式损坏 | "无法读取Excel文件，内容可能已损坏" | 1) 检查并修复文件 2) 使用备份文件 |
| 列数不足 | "Excel文件列数不足，需要7列数据" | 1) 检查并补充列 2) 使用正确的模板文件 |
| 数据为空 | "处理后的数据为空，请检查内容" | 1) 确认文件包含审查数据 2) 参考示例文件格式 |

## 错误类型详细说明

### 文件被占用

**诊断信息**: 脚本尝试读取文件时，发现该文件正被其他程序（Excel、文本编辑器等）打开或锁定。

**修复选项**:
1. 关闭占用该文件的程序后重新运行
2. 将文件复制一份，使用副本路径重新执行

### 权限不足

**诊断信息**: 脚本没有权限在指定的输出目录中创建文件或写入数据。

**修复选项**:
1. 以管理员身份运行程序
2. 指定一个当前用户有写入权限的输出目录

### 文件格式损坏

**诊断信息**: 脚本无法正常解析Excel或CSV文件，可能是文件扩展名不匹配或内容已损坏。

**修复选项**:
1. 使用Excel重新打开并保存文件，或使用备份文件
2. 确认文件是有效的Excel格式（.xlsx/.xls）而非网页导出文件

### 列数不足

**诊断信息**: Excel文件的列数少于7列，不满足脚本处理的最低要求。

**修复选项**:
1. 手动检查并补充缺失的列，确保有7列数据
2. 使用正确的模板文件重新生成数据

### 数据为空

**诊断信息**: 脚本读取文件后，发现有效数据区域为空或全为空白。

**修复选项**:
1. 确认文件包含实际的审查数据内容，非空表格
2. 参考示例文件格式，确保数据结构正确
